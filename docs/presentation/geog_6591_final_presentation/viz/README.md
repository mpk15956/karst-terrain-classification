# TDA Water-Level Filtration Visualization

Interactive Three.js visualization demonstrating persistent homology concepts through water-level filtration of terrain.

## Files

- `index.html` - Main HTML container
- `main.js` - Three.js scene logic and controls
- `style.css` - Styling
- `data/` - JSON data exports (generated by `export_viz_data.ipynb`)

## Data Format

### `hero_patch_dem.json`
2D nested array of normalized elevation values (0-1 range), shape: `[50][50]`

### `filtration_data.json`
Array of persistence features:
```json
{
  "id": 0,
  "dim": 1,
  "birth": 0.2,
  "death": 0.8,
  "type": "hole"
}
```

### `metadata.json`
Metadata including elevation min/max, patch ID, feature counts, etc.

## Usage

1. Run `notebooks/final_project/export_viz_data.ipynb` to generate data files
2. Open `index.html` in a browser (or embed via iframe in Quarto presentation)
3. Use slider to control water level and observe topological features

## Integration

Embedded in Quarto Reveal.js presentation via iframe:
```markdown
<iframe src="viz/index.html" width="100%" height="600px"></iframe>
```

## Troubleshooting

### Visualization shows flat terrain
- Verify DEM data was exported correctly
- Check browser console for errors (look for `[TDA_VIZ]:` prefix)
- Ensure `hero_patch_dem.json` contains 50×50 array
- Verify `metadata.json` has correct `patch_size` value

### Barcode diagram is empty
- Check `filtration_data.json` exists and contains features
- Verify persistence computation succeeded in export notebook
- Check browser console for data loading errors

### Iframe doesn't load in presentation
- Ensure `viz/` directory is in correct location relative to `presentation.html`
- Check browser console for CORS or file path errors
- Fallback static images should appear after 5 seconds
- Verify all three JSON files exist in `viz/data/` directory

### High-DPI display issues
- Barcode canvas should automatically scale for retina displays
- If text appears blurry, check browser zoom level
- Verify `window.devicePixelRatio` is detected correctly
